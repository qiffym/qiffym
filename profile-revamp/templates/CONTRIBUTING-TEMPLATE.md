# Contributing to {{PROJECT_NAME}}

Thanks for your interest in contributing! / Terima kasih atas minat Anda untuk berkontribusi!

## How to contribute

1. **Fork** the repository and create your branch from `main`:
   ```bash
   git checkout -b feature/your-feature
   ```
2. **Make your changes.** Keep them focused and small.
3. **Test** that the app still runs (see Quick Start in the README).
4. **Commit** using clear messages (Conventional Commits preferred):
   ```bash
   git commit -m "feat: add X"   # feat | fix | docs | refactor | chore
   ```
5. **Push** and open a **Pull Request** describing what and why.

## Reporting bugs

Open an issue and include:
- What you expected vs. what happened
- Steps to reproduce
- Environment (OS, PHP/Node version)
- Screenshots/logs if possible

## Code style

- Follow the existing style in the project.
- Run the project's formatter/linter before committing (e.g. `./vendor/bin/pint` for Laravel, `npm run lint` for JS/TS).

## Code of conduct

Be respectful and constructive. We want a welcoming project for everyone.
